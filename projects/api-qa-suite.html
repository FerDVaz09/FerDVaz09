🔧 ¿Qué es este proyecto?

Suite de pruebas enterprise-ready construida en TypeScript para máxima robustez y mantenibilidad. Arquitectura modular con un ApiClient reutilizable que abstrae las llamadas HTTP. Cubre autenticación JWT, operaciones CRUD completas y validación de contratos de API.

Incluye proyectos separados para tests de API pura y E2E de interfaz, con performance testing integrado y reporte modular. Integrado con GitHub Actions para ejecución automatizada en entornos de staging.

🎯 Características Principales

Typed Scalability

Uso de interfaces de TypeScript para garantizar que los payloads de API coincidan siempre con el esquema esperado.

JWT Auth Flow

Manejo automatizado de tokens de autenticación para flujos protegidos del lado del servidor.

Response Time Check

Assertions que validan que los tiempos de respuesta de la API estén dentro de los umbrales de SLA.

Modular ApiClient

Clase base reutilizable construida sobre Playwright Request para simplificar peticiones con headers y auth.

Contract Testing

Verificación de que los cambios en el backend no rompan la integración con el frontend o consumidores externos.

Detailed Debugging

Logs detallados de request/response adjuntos al reporte de errores para una resolución rápida.

🧪 Tecnologías Críticas

  • 📘 TypeScript: Tipado estricto para payloads y respuestas de API.
  • 🎭 Playwright: Runner principal para tests de API y orquestación E2E.
  • 🚀 Newman (Postman CLI): Integración de colecciones heredadas dentro del flujo CI/CD.
  • 🏗️ GitHub Actions: Pipeline de automatización con disparadores por PR.
