📋 Descripción del Proyecto

QA Log Sentinel es un sistema profesional de monitoreo visual de logs en tiempo real, diseñado con una interfaz tipo terminal/hacker inspirada en la estética Matrix. Esta herramienta permite a los QA Engineers simular, visualizar y analizar eventos del sistema como errores críticos, advertencias y mensajes informativos.

La aplicación utiliza Flask como backend para gestionar los logs en un archivo local, y un sistema de polling cada 2 segundos para actualizar la interfaz en tiempo real sin recargar la página. Perfecto para demos, testing de dashboards de monitoreo y entrenamiento en sistemas de logs.

⚡ Características Principales

📊 Monitoreo en Tiempo Real

Sistema de polling automático cada 2 segundos que consulta el servidor para obtener las últimas 15 líneas del log sin recargar la página.

🎨 Interfaz Matrix/Terminal

Diseño visual tipo CRT con fondo negro, texto verde fosforescente, efectos de scanlines y glow animado para una experiencia hacker auténtica.

🚨 Simulación de Eventos

Genera errores críticos, advertencias y mensajes INFO con un solo clic. Cada error incluye un ID único para tracking.

🎯 Colores Diferenciados

Sistema de colores inteligente: Rojo para [ERROR], Naranja para [WARNING], Cyan para [INFO] con efectos glow.

📈 Estadísticas Visuales

Panel de métricas en tiempo real que muestra total de logs, errores críticos, advertencias y mensajes informativos.

🔄 Auto-scroll Inteligente

El contenedor de logs se desplaza automáticamente al final cuando llegan nuevos mensajes, manteniendo siempre visible la actividad más reciente.

🛠️ Stack Tecnológico

Python 3.8+ Flask 3.0 HTML5 CSS3 JavaScript Vanilla Fetch API

💻 API Endpoints

Método Ruta Descripción
GET / Renderiza el dashboard principal
GET /get_logs Devuelve las últimas 15 líneas del log como JSON
POST /simulate_error Genera un error crítico con ID único (#DB1234, #MEM5678, etc.)
POST /simulate_info Genera un mensaje informativo (login, backup, sync, etc.)
POST /simulate_warning Genera una advertencia del sistema (CPU alto, memoria baja, etc.)
POST /clear_logs Limpia el archivo de logs con confirmación del usuario

💻 Ejemplo de Código: Sistema de Polling

// Función para obtener logs del servidor
async function fetchLogs() {
    try {
        const response = await fetch('/get_logs');
        const data = await response.json();
        
        if (data.logs) {
            logs = data.logs;
            renderLogs();
            updateStats();
        }
    } catch (error) {
        console.error('Error fetching logs:', error);
    }
}

// Polling: Actualizar logs cada 2 segundos
setInterval(fetchLogs, 2000);

// Renderizar logs con colores diferenciados
function renderLogs() {
    const container = document.getElementById('log-container');
    container.innerHTML = '';
    
    logs.forEach(log => {
        const logLine = document.createElement('div');
        logLine.className = 'log-line';
        
        // Aplicar clase según tipo de log
        if (log.includes('[ERROR]')) {
            logLine.classList.add('error');  // Rojo brillante
        } else if (log.includes('[WARNING]')) {
            logLine.classList.add('warning');  // Naranja
        } else if (log.includes('[INFO]')) {
            logLine.classList.add('info');  // Cyan
        }
        
        logLine.textContent = log;
        container.appendChild(logLine);
    });
    
    // Auto-scroll al final
    container.scrollTop = container.scrollHeight;
}
