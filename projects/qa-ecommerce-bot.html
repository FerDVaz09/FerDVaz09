🤖 QA E-Commerce Test Bot

Automatización End-to-End con Selenium, Page Object Model y Reportes Visuales

Python 3.8+ Selenium WebDriver Flask Page Object Model Faker

📋 Descripción del Proyecto

Bot de automatización QA profesional que simula un flujo completo de compra en SauceDemo.com, la plataforma estándar mundial para practicar testing de e-commerce. Este proyecto implementa las mejores prácticas de la industria: Page Object Model para código mantenible, esperas explícitas para estabilidad, y generación automática de reportes con evidencia visual de cada paso.

Proceder a Checkout

🧪 Suite de Pruebas QA

🟢 Nivel 1: Smoke Testing (Lo Vital)

  • TC-01: Flujo de Compra Exitoso (Happy Path)
    Objetivo: Verificar que un usuario puede comprar un producto de principio a fin.
    Pasos: Login con standard_user → Agregar "Sauce Labs Backpack" al carrito → Ir al carrito → Checkout → Llenar formulario → Finish.
    Validación: El mensaje final debe ser exactamente: "THANK YOU FOR YOUR ORDER".
  • TC-02: Login Fallido (Negative Testing)
    Objetivo: Verificar que el sistema bloquea intrusos.
    Pasos: Login con standard_user y password incorrecto123.
    Validación: Mensaje de error rojo: "Epic sadface: Username and password do not match any user in this service".

🟡 Nivel 2: Validaciones de Lógica (Functional Testing)

  • TC-03: Validación de Carrito Vacío
    Objetivo: Verificar que el contador del carrito funciona.
    Pasos: Login → Agregar 2 productos distintos → Verificar icono del carrito "2" → Remover 1 producto.
    Validación: El icono del carrito debe mostrar "1" tras remover.
  • TC-04: Validación de Formulario (Boundary Testing)
    Objetivo: Verificar que no se puede comprar sin datos de envío.
    Pasos: Login → Agregar producto → Checkout → Dejar "Zip/Postal Code" vacío → Continue.
    Validación: Error: "Error: Postal Code is required".

🔴 Nivel 3: QA Pro (Advanced Automation)

  • TC-05: Ordenamiento de Productos (Sorting Logic)
    Objetivo: Verificar que el filtro de precios funciona.
    Pasos: Login → Seleccionar "Price (low to high)" → Capturar precio del primer y último producto.
    Validación: El primer precio debe ser menor que el último (comparación float).
  • TC-06: Usuario Bloqueado (Security)
    Objetivo: Verificar el manejo de estados de usuario.
    Pasos: Login con locked_out_user y secret_sauce.
    Validación: Error: "Epic sadface: Sorry, this user has been locked out.".
  • TC-07: Cálculo de Impuestos (Math Validation)
    Objetivo: Verificar que la suma total es correcta.
    Pasos: Agregar producto de $29.99 → Checkout Overview → Extraer "Item total", "Tax" y "Total".
    Validación: (Item Total + Tax) == Total mostrado en pantalla.

💻 Ejemplo de Código: Page Object Model

Implementación profesional del patrón Page Object:

class LoginPage(BasePage):
    """Page Object para la página de Login"""
    
    USERNAME_INPUT = (By.ID, "user-name")
    PASSWORD_INPUT = (By.ID, "password")
    LOGIN_BUTTON = (By.ID, "login-button")
    
    def login(self, username, password):
        """Realiza el login con esperas explícitas"""
        self.enter_text(*self.USERNAME_INPUT, username)
        self.enter_text(*self.PASSWORD_INPUT, password)
        self.click_element(*self.LOGIN_BUTTON)
    
    def is_login_successful(self):
        """Verifica si el login fue exitoso"""
        try:
            self.wait.until(EC.url_contains("/inventory.html"))
            return True
        except TimeoutException:
            return False

Beneficios de esta arquitectura: Código reutilizable, fácil de mantener, y permite cambios en la UI sin reescribir toda la lógica de prueba.
