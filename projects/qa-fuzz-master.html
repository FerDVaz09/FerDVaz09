📋 Descripción del Proyecto

QA Fuzz Master es una aplicación web profesional diseñada para realizar pruebas de seguridad automatizadas mediante técnicas de fuzzing. Esta herramienta permite a los testers de QA identificar vulnerabilidades comunes en aplicaciones web como inyecciones SQL, desbordamientos de buffer y ataques XSS (Cross-Site Scripting).

La aplicación utiliza Selenium WebDriver para automatizar la interacción con formularios web, inyectando payloads maliciosos generados con Faker y capturando evidencia visual mediante screenshots automáticos.

⚡ Características Principales

🔒 Pruebas de Inyección SQL

Detecta vulnerabilidades de SQL Injection mediante payloads como ' OR '1'='1 que intentan comprometer la seguridad de bases de datos.

💥 Buffer Overflow Testing

Genera cadenas de 500+ caracteres para identificar problemas de validación de longitud en campos de entrada.

🎭 Cross-Site Scripting (XSS)

Inyecta scripts maliciosos y emojis especiales para probar filtros de validación y escapado de HTML.

📸 Captura Automática

Genera screenshots antes y después de cada prueba para documentar el comportamiento de la aplicación bajo test.

🎨 Interfaz Cyberpunk

Diseño visual inspirado en la estética hacker con efectos de glitch y animaciones futuristas.

⚙️ Selenium WebDriver

Automatización browser-level con Chrome en modo headless para ejecución rápida y eficiente.

🛠️ Stack Tecnológico

Python 3.8+ Flask Selenium WebDriver Chrome Driver Faker HTML5 / CSS3 JavaScript

🚀 Cómo Usar la Herramienta

  1. Clona el repositorio e instala dependencias: 
    # Clona el proyecto
git clone https://github.com/FerDVaz09/QA-Fuzz-Master.git
cd QA-Fuzz-Master

# Instala las dependencias
pip install -r requirements.txt
  2. Descarga Chrome Driver:

    Descarga el driver compatible con tu versión de Chrome desde chromedriver.chromium.org y colócalo en la carpeta del proyecto.

  3. Ejecuta la aplicación Flask: 
    python app.py

# La aplicación estará disponible en:
http://localhost:5000
  4. Configura el objetivo:

    En la interfaz web, introduce la URL del sitio que deseas testear. Asegúrate de tener permiso para realizar pruebas de seguridad.

  5. Selecciona el tipo de prueba:
    • SQL Injection: Detecta vulnerabilidades de inyección SQL
    • Buffer Overflow: Prueba límites de validación de longitud
    • XSS (Cross-Site Scripting): Identifica problemas de sanitización
  6. Ejecuta el ataque y analiza resultados:

    La herramienta ejecutará las pruebas automáticamente, capturará screenshots y mostrará un reporte con el estado de cada prueba y las evidencias visuales.

💻 Ejemplo de Código: Función de SQL Injection

# Función para probar inyección SQL
def test_sql_injection(driver, url):
    results = []
    
    # Payloads de inyección SQL
    sql_payloads = [
        "' OR '1'='1",
        "admin'--",
        "' OR 1=1--",
        "'; DROP TABLE users--"
    ]
    
    driver.get(url)
    time.sleep(2)
    
    # Captura screenshot inicial
    screenshot_before = f"sql_injection_before_{datetime.now().strftime('%Y%m%d_%H%M%S')}.png"
    driver.save_screenshot(f"static/screenshots/{screenshot_before}")
    
    # Busca y rellena inputs con payloads
    inputs = driver.find_elements(By.TAG_NAME, "input")
    
    for i, input_field in enumerate(inputs):
        if i < len(sql_payloads):
            try:
                input_field.clear()
                input_field.send_keys(sql_payloads[i])
            except:
                pass
    
    time.sleep(1)
    
    # Captura screenshot después del ataque
    screenshot_after = f"sql_injection_after_{datetime.now().strftime('%Y%m%d_%H%M%S')}.png"
    driver.save_screenshot(f"static/screenshots/{screenshot_after}")
    
    results.append({
        'type': 'SQL Injection',
        'payload': "' OR '1'='1",
        'status': 'Completado',
        'screenshot_before': screenshot_before,
        'screenshot_after': screenshot_after
    })
    
    return results

📊 Tipos de Pruebas y Payloads
