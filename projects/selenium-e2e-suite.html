🔧 ¿Qué es este proyecto?

Suite completa de pruebas end-to-end para automatización de navegadores web, construida en Python con Selenium y Pytest. Implementa el patrón Page Object Model (POM) para separar la lógica de los tests de la interacción con la interfaz, lo que facilita el mantenimiento y la escalabilidad del framework.

Incluye gestión de fixtures con conftest.py, datos de prueba centralizados en un directorio dedicado, y un pipeline CI/CD con GitHub Actions que ejecuta la suite de forma automática ante cada cambio en el repositorio.

🎯 Características Principales

Page Object Model

Separación clara entre la lógica de prueba y los selectores de UI, haciendo el código reutilizable y fácil de mantener ante cambios en la interfaz.

Pytest Framework

Aprovecha fixtures, parametrización y plugins de Pytest para tests expresivos, con reportes detallados de resultados y fallas.

Test Data Management

Datos de prueba centralizados en un directorio /data, desacoplados de la lógica de test para facilitar su actualización y mantenimiento.

CI/CD con GitHub Actions

Pipeline automatizado que ejecuta toda la suite en cada push o pull request, detectando regresiones antes de que lleguen a producción.

Configuración Centralizada

conftest.py y pytest.ini para manejar configuraciones globales, entornos y fixtures reutilizables en todos los módulos de prueba.

Estructura Modular

Organización en directorios /pages, /tests y /data que facilita escalar el proyecto con nuevos módulos sin romper lo existente.

🧪 Tecnologías Críticas

  • 🐍 Python: Lenguaje principal para máxima legibilidad y ecosistema QA maduro.
  • 🌐 Selenium WebDriver: Automatización cross-browser de interacciones de usuario en la UI.
  • 🧪 Pytest: Framework de testing con fixtures, parametrización y plugins de reporte.
  • 📐 Page Object Model: Patrón de diseño para abstraer selectores y acciones por página.
  • 🏗️ GitHub Actions: Ejecución automática de tests en entornos de integración continua.
